## Deviations (need your eyes)

1. **whisper-small.en q8_0, not q4_0.** Upstream ggerganov/whisper.cpp no
   longer hosts q4_0 (only q5_1/q8_0/f32); the phase-1 prompt's URL 404s.
   Also the ChristianAzinn MiniLM URL 401s — used
   second-state/All-MiniLM-L6-v2-Embedding-GGUF Q4_0 instead. q8_0 chosen
   over q5_1 on measurement: 5.3s vs 12.1s on the same clip (i8mm).
   D10-style factual correction candidates.
2. **OCR baseline = rec model + numpy projection segmentation.** No det
   model: DB postprocessing needs opencv/pyclipper, neither available on
   bionic (no wheels; not in termux main). PP-OCRv5 English rec ONNX
   (7.8MB, xberg-io/paddleocr-onnx-models, Apache-2.0) + Otsu binarize +
   row-projection line bands. Good on screenshots/documents; det-model
   upgrade path documented in npu/ocr_engine.py docstring.
3. **AUDIO_MAX_SEC=600 cap** (settings.py): whisper windows internally, so
   the 30s "max_context" is not a request cap; 600s guards runaway batch.
   My call — veto if you want a different number.
4. **classify+llm share one llama-server (:8463)** — same weights, one
   2GB residency instead of two.

## Platform facts for memd

- big.LITTLE thread pinning is load-bearing: whisper 11s clip — 12s @ -t4,
  81s @ -t6, 137s @ -t8. llama-server default threads made a 4-token
  completion take 30s; `-t 4 -tb 4` → 430ms TTFT. Everything pinned to 4.
- llama-server binds its port before the model loads and 503s meanwhile —
  readiness = GET /health 200, not port-open (npu/models.py).
- Termux cmake under proot needs `PREFIX` env exported or it dies reading
  `$PREFIX/include/android/api-level.h` (CMakeDetermineSystem patch).
- tar hardlink extraction fails under proot (link2symlink) AND on the
  scratchpad fs; python tarfile falls back to copying — use it for .debs.
- xberg-io OCR dict: line 0 is "#" = CTC blank placeholder; rec model
  output is already softmaxed (don't re-softmax for confidence).
